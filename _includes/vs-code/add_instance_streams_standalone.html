An IBM Streams instance is required. If you need to install an instance, start here and follow the instructions.

To add an instance in VS Code, either:

  • Bring up the Streams Explorer view by clicking on the Streams icon on the left side of VS Code. Then, click on the + button.
  • Run the IBM Streams: Add Instance command.

You must provide the following information in order for this extension to connect to your instance.

  • IBM Streams instance REST service URL: IBM Streams provides a REST API that can be used to retrieve information about the instance. This service has a URL associated with it. For example, when the service is exposed as a node port: https://123.45.67.89:30001.
  • IBM Streams instance build service URL (optional): IBM Streams provides an optional build service for compiling and building Streams applications. This service has a URL associated with it. For example, when the service is exposed as a node port: https://123.45.67.89:30002. If not specified, you will not be able to build your Streams applications.
  • IBM Streams instance Console URL (optional): IBM Streams includes an integrated console that you can use to view the health of the instance and its applications. You can manage instances and resources, configure security, and monitor jobs from a single location. If not specified, the Console URL will be retrieved from the Streams REST API.
  • IBM Streams instance credentials: These are the credentials (username and password) used to authenticate to the IBM Streams instance.

Once you add the instance, you can view the instance by clicking on the IBM Streams icon on the left side to bring up the Streams Explorer view.
